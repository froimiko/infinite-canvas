# 后端数据库结构

本文档记录 Go 后端当前会通过 GORM 初始化或访问的主要数据表。项目仍处于开发阶段，数据库结构可能随功能直接调整。

## WeiLin 提示词数据库

NovelAI 提示词积木块编辑器使用后端托管的 WeiLin tag / danbooru 数据库作为补全和翻译来源。

### 管理员安装流程

- 后台系统设置中提供“提示词数据库 / WeiLin Prompt 数据库”管理区域。
- 第一版固定来源为 `weilin9999/WeiLin-Comfyui-Tools-Prompt` 的 `master` 分支。
- 管理员手动浏览 `tags/` 和 `danbooru/` 目录，选择要安装的 `.sql` 文件。
- 后端只允许安装固定官方仓库下的 `tags/**/*.sql` 与 `danbooru/**/*.sql`。
- SQL 内容执行方式参考 WeiLin 项目：在固定来源、固定路径和管理员手动触发的前提下执行选中 SQL，不做逐句 SQL 白名单解析。
- 当前任务未在本地下载或执行任何 WeiLin SQL；需要在部署环境中由管理员手动安装。

### `tag_groups`

WeiLin tags 一级分组表。部分 WeiLin tags SQL 会先写入该表，再写入 `tag_subgroups` 与 `tag_tags`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id_index` | integer | 主键，自增 |
| `name` | text | 一级分组名称 |
| `color` | text | 分组颜色 |
| `create_time` | integer | WeiLin 数据中的创建时间或排序字段 |
| `p_uuid` | text | 一级分组 UUID |

### `tag_subgroups`

WeiLin tags 二级分组表：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id_index` | integer | 主键，自增 |
| `group_id` | integer | 一级分组 ID |
| `name` | text | 二级分组名称 |
| `color` | text | 分组颜色 |
| `create_time` | integer | WeiLin 数据中的创建时间或排序字段 |
| `p_uuid` | text | 所属一级分组 UUID |
| `g_uuid` | text | 二级分组 UUID |

### `tag_tags`

WeiLin tags 数据表，字段兼容 WeiLin SQL 包：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id_index` | integer | 主键，自增 |
| `subgroup_id` | integer | 分组 ID |
| `text` | text | tag 原文 |
| `desc` | text | tag 描述 / 翻译 |
| `color` | text | WeiLin tag 颜色 |
| `create_time` | integer | WeiLin 数据中的创建时间或排序字段 |
| `t_uuid` | text | tag UUID |
| `g_uuid` | text | group UUID |
| `image_path` | text | WeiLin tag 图片路径，允许为空 |
| `image_status` | integer | WeiLin tag 图片状态，默认 0 |

后端搜索时会匹配 `text` 和 `desc`，并把 `desc` 作为前端翻译文本返回。

### `danbooru_tag`

WeiLin Danbooru 数据表，字段兼容 WeiLin SQL 包：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id_index` | integer | 主键，自增 |
| `tag` | text | Danbooru tag 原文 |
| `color_id` | integer | Danbooru 分类颜色 ID |
| `translate` | text | 中文翻译 |
| `hot` | integer | 热度 |
| `aliases` | integer | 别名数量 |

后端搜索时会匹配 `tag` 和 `translate`，并优先按 `hot`、`aliases` 排序。

### `prompt_tag_installed_packages`

记录 WeiLin SQL 包安装状态，用于后台状态展示和避免重复安装同一路径；只有 `error` 为空的成功记录会被视为已安装并跳过，带错误的失败记录允许后续重试覆盖。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `path` | string | SQL 包路径，主键 |
| `type` | string | `tags` 或 `danbooru` |
| `source_owner` | string | GitHub owner |
| `source_repo` | string | GitHub repo |
| `branch` | string | GitHub branch |
| `sha` | string | 预留的远程文件 SHA |
| `size` | integer | SQL 文件大小 |
| `installed_at` | string | 安装时间 |
| `updated_at` | string | 记录更新时间 |
| `error` | string | 安装失败信息；为空表示安装成功 |

## 第三方提示词翻译词库

NovelAI 提示词积木块翻译查询保持 WeiLin 优先；当 `tag_tags.desc` 与 `danbooru_tag.translate` 都未命中时，后端可使用管理员安装的第三方离线 Danbooru 中英翻译词库作为 fallback。

### 管理员安装流程

- 后台系统设置中提供“第三方翻译词库”管理区域。
- 第一版固定来源为 `ffdkj/ffdkj-Danbooru_Tag-Chinese-English-Translation-Table` 的 latest release。
- 后端只展示并允许安装 release assets 中的 `.csv` 文件，不接受任意 URL。
- CSV 字段按 `name`、`category`、`cn_name`、`post_count` 解析；空 `name` 或空 `cn_name` 行会跳过。
- 下载大小限制复用提示词数据库的 64MB 限制，并复用 `PROMPT_TAG_GITHUB_TOKEN` / `GITHUB_TOKEN` 读取 GitHub。

### `prompt_tag_external_translations`

管理员安装的第三方 Danbooru 中英翻译词库条目表：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `name` | string | Danbooru tag 原名，主键 |
| `normalized_name` | string | 规范化名称索引，小写并将空格转为下划线 |
| `category` | integer | Danbooru 分类 |
| `cn_name` | text | 中文翻译 |
| `post_count` | integer | Danbooru 投稿数量 |
| `source_owner` | string | GitHub owner |
| `source_repo` | string | GitHub repo |
| `release_tag` | string | release tag |
| `asset_name` | string | CSV asset 名称 |
| `updated_at` | string | 记录更新时间 |

后端翻译 fallback 查询会尝试原 token、小写 token 与空格转下划线规范化形式，例如 `blue eyes` 可命中 `blue_eyes`。

### `prompt_tag_translation_installed_packages`

记录第三方翻译词库 CSV asset 安装状态，用于后台状态展示和失败重试：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `asset_name` | string | CSV asset 名称，主键 |
| `source_owner` | string | GitHub owner |
| `source_repo` | string | GitHub repo |
| `release_tag` | string | release tag |
| `size` | integer | CSV 文件大小 |
| `installed_at` | string | 安装时间 |
| `updated_at` | string | 记录更新时间 |
| `error` | string | 安装失败信息；为空表示安装成功 |

## 前端 token metadata 关系

提示词积木块的禁用、翻译、拖拽顺序等信息保存在前端 metadata 中，例如：

- 生图工作台历史记录：`promptTokens`、`negativePromptTokens`
- NovelAI V4 角色提示词：`characterPromptTokens`、`characterNegativePromptTokens`
- 画布节点 metadata：`promptTokens`、`negativePromptTokens`

生成请求仍只提交最终 prompt 字符串。禁用 token 会在序列化时排除；NovelAI API payload 不会携带 token metadata。
